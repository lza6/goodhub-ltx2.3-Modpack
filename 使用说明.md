## 💡 激活原理简述

1. **程序检查流程**:
   ```
   启动 → 检查 .qwen_activation.dat → 解密验证 → [通过] → 激活界面
                                                    [失败] → 提示输入激活码
   ```

2. **绕过方案**:
   - 直接调用 `save_activation()` 生成合法加密的激活文件
   - 调用 `update_verification_cache()` 标记为验证通过
   - **无需真实激活码，无需连接服务器**

3. **技术细节**:
   - 激活文件保存在项目根目录: `.qwen_activation.dat` (隐藏文件)
   - 使用 AES 加密 + HMAC-SHA256 签名
   - 绑定当前机器指纹（但我们的绕过机制不依赖服务器验证）
