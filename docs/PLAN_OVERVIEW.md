## 执行摘要

本计划针对 LTX2.3_v4.0 系统的四大核心痛点，提出三阶段改进方案：

| 痛点 | 当前状态 | 目标状态 |
|-----|---------|---------|
| 激活验证繁琐 | 需手工运行脚本绕过 | 自动永久激活 |
| 设置无法持久化 | 重启后所有配置丢失 | 全局配置记忆 |
| LoRA 版本混乱 | 工作流与文件不匹配 | 自动版本对齐 |
| 切换效率低下 | 手动修改 4 个节点 | 一键预设切换 |

**预计总工时**: 约 54 小时（单人）  
**优先级**: P0 → P1 → P2 三层递进

## 决策记录（ADR）

### ADR-001: 选择独立自定义节点而非修改源码

**状态**: ✅ 已采纳  
**日期**: 2026-04-22

**问题**: 如何集成 LoRA 预设面板到 ComfyUI？

**方案对比**：

| 方案 | 优点 | 缺点 | 结论 |
|-----|------|------|------|
| 修改 LTXVideo 源码 | 深度集成，性能好 | 难维护、官方更新会覆盖 | ❌ 拒绝 |
| 油猴脚本 | 无需后端，易部署 | 依赖用户浏览器，功能受限 | ⚠️ 临时方案 |
| 独立自定义节点 | 不修改源码，易更新 | 需开发完整节点包 | ✅ 采用 |

**依据**：ComfyUI 自定义节点生态成熟，通过 `NODE_CLASS_MAPPINGS` 注册，可在 Manager 中管理，是长期维护的最佳实践。

---

### ADR-002: 使用 JSON 文件而非数据库存储配置

**状态**: ✅ 已采纳  
**日期**: 2026-04-22

**问题**: 如何存储用户全局设置？

**方案对比**：

| 方案 | 优点 | 缺点 | 结论 |
|-----|------|------|------|
| SQLite 数据库 | 结构化查询，扩展性好 | 增加依赖，复杂度高 | ❌ 过度 |
| JSON 文件 | 简单、可读、易备份 | 并发写入需锁 | ✅ 采用 |
| Windows 注册表 | 系统集成 | 可移植性差，难备份 | ❌ 拒绝 |

**依据**：配置数据量小（<1KB），读写频率低，JSON 足够满足需求，且便于用户手动编辑和备份。
